# v0.1.0 多平台兼容 + 首次引导完善计划

## 核心问题

审计发现 **onboarding 向导是断的**：它写入 credentials 但从不写入 config.toml 的 `provider_id`/`model`/`base_url`，导致 `build_chat_model` 走错误的 S0 fallback 路径，用户刚输入的 key 根本没被读到。首次运行必然失败。

---

## 修复 1：Onboarding 向导持久化配置（CRITICAL）

**根因**：`run_onboarding` 返回 `OnboardingResult(provider_id, model, base_url, ...)`，但 `_maybe_onboard` 丢弃了返回值，从不写入 config.toml。

**改动**：
- **`src/coderio/cli/onboarding.py`**：`run_onboarding` 在写 credentials 后，同时把 `provider_id`、`model`、`base_url`（如果是 openai_custom）写入 `~/.coderio/config.toml` 的 `[model]` section。用 tomllib 读 + tomli-w 写的方式合并更新（不覆盖用户已有的其他 section）。
- **`src/coderio/config/bootstrap.py`**：把 `_SAMPLE_CONFIG` 的 `[model]` section 改为**不含 default/provider/base_url**（这些由 onboarding 填入），避免 sample config 的旧值（glm-4.5/paas/v4）覆盖 onboarding 的选择。

## 修复 2：Onboarding 跳过逻辑优化

**问题**：`_maybe_onboard` 只检查 credentials 文件，用户设了 `ANTHROPIC_API_KEY` 环境变量仍被强制走向导。

**改动**：
- **`src/coderio/cli/repl.py` `_maybe_onboard`**：如果 credentials 有内容 **或** `config.toml` 已有 `provider_id` **或** 环境变量 `ANTHROPIC_API_KEY`/`OPENAI_API_KEY` 存在，则跳过向导。

## 修复 3：新增国际 provider（OpenAI / Anthropic / Ollama）

**问题**：provider 列表只有智谱/阶跃，国际用户没有可选项。

**改动**：
- **`src/coderio/cli/providers.py`**：新增 3 个 provider：
  - `openai` — OpenAI 直连（`api.openai.com/v1`，gpt-4o/gpt-4.1-mini 等）
  - `anthropic` — Anthropic 直连（`api.anthropic.com`，claude-sonnet/haiku）
  - `ollama` — 本地 Ollama（`http://localhost:11434/v1`，无需 key）
- onboarding 菜单分组调整：Coding Plan / API Key 直连 / 国际 / 本地

## 修复 4：文档多平台命令

**问题**：README/CONTRIBUTING 的运行/测试命令只有 `.venv/Scripts/python.exe`（Windows），Linux/macOS 用户无法复制。

**改动**：
- **README.md**：运行/测试 section 加 Linux/macOS 的 `.venv/bin/python` 变体
- **CONTRIBUTING.md**：同上
- **`scripts/verify_*.py` docstrings**：加双平台示例

## 修复 5：bash 工具错误消息跨平台

**改动**：
- **`src/coderio/tools/bash.py:41`**：`FileNotFoundError` 消息按 `sys.platform` 分支——Windows 提示装 Git Bash，Linux/macOS 提示装 bash 包。
