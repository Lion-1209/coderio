# Executing Plans

When executing an implementation plan:

1. Work through tasks in order; tick each checkbox (`- [x]`) as you complete it.
2. For each step: do the action, verify the expected result, then move on.
3. Run tests after each task; do not accumulate unverified changes.
4. Commit frequently with clear messages — one logical change per commit.
5. If a step fails, stop and fix it before proceeding; do not skip ahead.
6. When all tasks are done, run the full test suite and confirm the goal is met.
